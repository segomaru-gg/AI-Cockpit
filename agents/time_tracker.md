# Time Tracker Agent

## トリガー
ユーザーが以下のコマンドを発行したとき、このプロトコルに従って動作する:

| コマンド | 動作 |
|---|---|
| `S: [タスク名]` | タスク開始を記録 |
| `E: [備考]` | タスク終了を記録・所要時間を計算 |
| `Log Time` | 今日のタイムログを表示 |

---

## 記録先
`domains/business/cloud/timelog/YYYY-MM-DD.md`（当日の日付）

---

## S: [タスク名] の処理手順

1. 現在時刻（JST）を取得: `date '+%H:%M'`
2. 今日のタイムログファイルを開く（なければ作成）
3. 新しい行を追記:
   ```
   | {N} | {タスク名} | {HH:MM} | - | - | 作業中... |
   ```
4. ユーザーに返答:
   ```
   ⏱ スタート: {タスク名} [{HH:MM}]
   ```

---

## E: [備考] の処理手順

1. 現在時刻（JST）を取得: `date '+%H:%M'`
2. 今日のタイムログファイルを開く
3. 最後の「作業中...」行を探し、終了時刻・所要時間・備考を埋める:
   ```
   | {N} | {タスク名} | {開始} | {HH:MM} | {X}min | {備考} |
   ```
4. ファイル末尾の「合計」行を更新
5. ユーザーに返答:
   ```
   ✅ 完了: {タスク名} [{開始} → {終了}] {X}min
   本日累計: {累計}min
   ```

---

## タイムログファイルのフォーマット

```markdown
# CLOUD タイムログ YYYY-MM-DD

| # | タスク | 開始 | 終了 | 時間 | 備考 |
|---|---|---|---|---|---|
| 1 | タスク名 | HH:MM | HH:MM | Xmin | ... |

---
**本日合計**: Xmin（X.Xh）
```

---

## ルール
- 時刻は常に JST（`date '+%H:%M'` で取得）
- `S:` の後に前のタスクが「作業中...」のままなら、ユーザーに確認してから上書き
- `E:` 備考が空の場合は「-」を入力
- 1日の終わりに `journal/` への転記は不要（タイムログがそのまま記録として機能）
