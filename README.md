# AI-Cockpit: The Blueprint Life Operating System

[日本語による説明は後半にあります](#japanese-version)

This is a unified workspace designed to manage all domains of life using the **Blueprint Protocol**. It integrates business, operations, personal learning, and brand evolution into a single, cohesive "Life Operating System."

## 🌌 Core Concept: Life OS

The **AI-Cockpit** is more than just a code repository; it is a tactical control center. By standardizing project metadata and task management, it allows for a high-level overview of progress across multiple independent domains.

## 🏗 Project Structure

- **[BluePrint/](file:///Users/ryosaigo/AI-Cockpit/BluePrint)**: The Master Control Dashboard. An Astro-based interface that aggregates tasks and journal entries from all sibling projects.
- **[aeternum/](file:///Users/ryosaigo/AI-Cockpit/aeternum)**: Personal Brand & Portfolio Evolution.
- **[Oduman/](file:///Users/ryosaigo/AI-Cockpit/Oduman)**: Business Strategy & Consulting.
- **[BoaBASE/](file:///Users/ryosaigo/AI-Cockpit/BoaBASE)**: Operations & Management.
- **[TOEIC/](file:///Users/ryosaigo/AI-Cockpit/TOEIC)**: Personal Growth & Language Learning.

## 📜 Blueprint Protocol v1.0

Every project in this workspace follows a standardized structure to ensure compatibility with the Master Cockpit:

1. **`blueprint.yaml`**: Metadata (Name, ID, Category, Status).
2. **`tasks/`**: Domain-specific tasks stored as Markdown with technical frontmatter.
3. **`journal/`**: Immutable logs of decisions and milestones.
4. **`.cursorrules`**: Global agent instructions for protocol enforcement.

## 🚀 Getting Started

To launch the Master Control Dashboard:

```bash
cd BluePrint
npm install
npm run dev
```

The dashboard will be available at `http://localhost:4322/`.

# AI-Cockpit: Blueprint Life Operating System (日本語)

これは **Blueprint Protocol** を用いて、人生の全領域を管理するために設計された統合ワークスペースです。ビジネス、実務（Ops）、個人学習、そしてブランド進化を一つの首尾一貫した「Life Operating System」として統合します。

## 🌌 コア・コンセプト: Life OS

**AI-Cockpit** は単なるコード・リポジトリではありません。それは「戦術管制センター」です。プロジェクトのメタデータとタスク管理を標準化することで、複数の独立したドメインにわたる進捗を俯瞰して把握することを可能にします。

## 🏗 プロジェクト構成

- **[BluePrint/](file:///Users/ryosaigo/AI-Cockpit/BluePrint)**: マスター・コントロール・ダッシュボード。全プロジェクトのタスクとジャーナルを集約して表示する、Astro製のインターフェース。
- **[aeternum/](file:///Users/ryosaigo/AI-Cockpit/aeternum)**: パーソナルブランドとポートフォリオの進化。
- **[Oduman/](file:///Users/ryosaigo/AI-Cockpit/Oduman)**: ビジネス戦略とコンサルティング。
- **[BoaBASE/](file:///Users/ryosaigo/AI-Cockpit/BoaBASE)**: 実務・運用管理。
- **[TOEIC/](file:///Users/ryosaigo/AI-Cockpit/TOEIC)**: 自己研鑽・語学学習。

## 📜 Blueprint Protocol v1.0

このワークスペース内の各プロジェクトは、マスター・コックピットとの互換性を確保するために以下の標準構造に従います：

1. **`blueprint.yaml`**: メタデータ（名前、ID、カテゴリ、ステータス）。
2. **`tasks/`**: 各ドメインのタスク。技術的なフロントマターを備えた Markdown 形式で管理。
3. **`journal/`**: 意思決定やマイルストーンを記録する、改変不能なログ。
4. **`.cursorrules`**: プロトコルを遵守させるための、エージェント向け共通指示書。

## 🚀 始め方

マスター・コントロール・ダッシュボードを起動するには：

```bash
cd BluePrint
npm install
npm run dev
```

ダッシュボードは `http://localhost:4322/` でアクセス可能です。
