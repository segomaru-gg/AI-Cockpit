# 2026-02-16: Project Dashboard Navigation - 現場へのダイレクトアクセス

マスターコックピットから各プロジェクト独自の詳細ダッシュボードへ直接遷移できる機能を実装した。

### 実施内容：
1.  **ダッシュボード自動検出**:
    - `projects.ts` を拡張し、プロジェクトフォルダ内の `dashboard/index.html` を自動的にダッシュボードURLとして認識するロジックを実装。
    - `blueprint.yaml` での `dashboard_url` 指定（オーバーライド）にも対応。
2.  **ローカルサービングの解決**:
    - Astro の dev サーバが別フォルダの HTML を直接開けない制約を解決するため、`public/projects_root` に各 PJ へのシンボリックリンクを作成。
    - 当初発生した再帰的リンクによるサーバクラッシュ問題を、個別リンク作成方式に切り替えることで解決。
3.  **UI/UX の洗練**:
    - プロジェクト名をリンク化し、ホバー時に `↗` アイコンが表示される等、直感的かつテクニカルな操作感を実現。

### 考察：
単なる「状況確認」だけでなく、ワンクリックで「現場（個別ダッシュボード）」に飛び、詳細な操作や分析を開始できるようになった。これにより、Cockpit が真の意味でマルチプロジェクトの中枢として機能し始めた。
