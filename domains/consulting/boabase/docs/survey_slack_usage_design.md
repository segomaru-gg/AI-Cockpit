# Slack導入に向けた現況調査 (Communication Audit)

## 1. 目的
- 現在のLINE/DM中心のコミュニケーションによる「見えないコスト（精神的負荷、情報紛失、意思決定の遅れ）」を**数値化**する。
- Slack導入の目的を「ツール変更」ではなく「業務OSのアップデート」と定義付けるための根拠を集める。

## 2. 調査対象
- オーナー (Yanagi)
- 店長/スタッフ (Mizuno, Kei, Others)
- 外部連携者 (Takada, etc.)

## 3. 調査項目 (Draft)

### A. 定量データ (頻度・量)
1. **通知の数:** 1日平均何件の業務LINE通知が来ますか？
   - [ ] 0-10件
   - [ ] 10-30件
   - [ ] 30-50件
   - [ ] 50件以上
2. **DM比率:** 業務連絡の何割が「個人宛DM」で来ますか？
   - [ ] ほぼ全てグループ
   - [ ] 半々くらい
   - [ ] ほぼDM
3. **即レス圧力:** 「5分以内に返信しないと」というプレッシャーを感じる頻度は？
   - [ ] 常に感じる
   - [ ] たまに感じる
   - [ ] 感じない

### B. 定性データ (課題感)
4. **情報の検索:** 「あの件どうなったっけ？」と過去ログを遡るのにかかる時間は？
   - [ ] すぐ見つかる
   - [ ] 5分くらい探す
   - [ ] 諦めて聞くことが多い
5. **公私混同:** プライベートのLINEに業務連絡が紛れ込むことのストレス度は？ (1-5段階)
6. **決定プロセスの透明性:** 知らない間に重要なことが決まっていた経験はありますか？

## 4. 実施方法
- **形式:** Google Forms (匿名回答可) or ヒアリングシート配布
- **期間:** 2月末までに実施・回収
- **ゴール:** これらを「Communication Dept (負債)」として可視化し、3月のSlack導入ロードマップに乗せる。
