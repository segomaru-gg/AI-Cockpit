# BoaBASE システムコンテキスト

## 1. ミッションとビジョン
- **コアミッション:** 「CHILL & Community」の創造。
- **ビジョン:** シーシャを単なる喫煙ではなく、アート、音楽、空間と融合した「素敵な文化」「終わりのない旅」として広め、人々のサードプレイス（居場所）をつくる。

## 2. ロードマップとフェーズ

### フェーズ1：店長業務（運営実務期）
- **期間:** 2026年3月まで
- **役割:** 店長（西郷 亮）
- **主要目的:** 六甲店と元町店のオペレーションの卓越、コミュニティの醸成、および文化経営への移行準備。
- **重要タスク:**
    - コンセプト主導のサービス維持（70種類以上のフレーバー、自家製ドリンク）。
    - 「Boa WALL」のキュレーション（月替わりのアート展示）。
    - **移行準備（最優先）:** 情報インフラ（Slack/Drive）の再設計、文化的な「暗黙知」の言語化。

### フェーズ2：文化経営監督（ガバナンス・戦略期）
- **期間:** 2026年4月以降
- **役割:** 文化経営監督（Aeternum / 外部パートナー）
- **主要目的:** 戦略的監督、ブランド翻訳、確立された「Concept Driven」経営モデルの実装。
- **重要タスク:**
    - 月次文化ガバナンス（文化監督としての関与）。
    - 外部チャネル（note, YouTube, ZINE）を通じたブランド翻訳。
    - 「意味（思想）→ 構造（仕組み）→ 表現（アウトプット）」のサイクルの管理。

## 3. マルチエージェント運用における Antigravity の役割
本プロジェクトにおいて、Antigravity (Caesar) は **「実行エージェント」** として機能する。Segoマルチエージェントプロトコルに基づき、以下の役割に特化する。

- **役割（実行・構築・推進）:**
    - **ロードマップの具体的実装:** ChatGPTが設計しGeminiが判断した戦略を、具体的なタスクおよび手順へと落とし込み、構築する。
    - **インフラの技術的構築:** Slack/Drive等の環境整備、ドキュメントの構造化、ツールの実装。
    - **タスク実行の加速:** 現場で発生した課題に対し、具体的な「どう作るか/どう動かすか」の手順を提示し、実行を支援する。
    - **進捗管理ツール（Dashboard）の維持:** `tasks/` および `journal/` を適切に運用し、推進力を維持する。

- **協働方針:**
    - **思考（ChatGPT）の尊重:** 「なぜ」「何を」といった意味解釈や哲学については、ChatGPTのコンテキストを最上位として参照し、Antigravity自身が勝手に意味付けを行わない。
    - **判断（Gemini）の尊重:** 実行の優先順位や現実的な選択については、Geminiの合理的な判断に従う。
    - **記録（NotebookLM）への橋渡し:** 実行結果やプロセスを適切に整理し、NotebookLMに蓄積可能な形式で出力する。

## 4. プロジェクト体制
- **運営（Running）:** 合同会社holonical（採用、シフト、実務マニュアルを担当）。
- **監督（Curation）:** Aeternum（文化的整合性、戦略的立案、ブランドコミュニケーションを担当）。
- **実行支援（Execution）:** Antigravity（本ワークスペース）。
