# 返金ステータス Discord 通知 Bot — セットアップ手順

Python・Google Cloud Console・認証ファイル、すべて不要です。
スプレッドシートだけで完結します。

---

## STEP 1 — Discord Webhook URL を取得する

1. Discord で通知を送りたいチャンネルを開く
2. チャンネル名を **右クリック** → 「チャンネルの編集」
3. 左メニュー「**連携サービス**」→「**ウェブフック**」→「**新しいウェブフック**」
4. 名前を設定（例: `返金Bot`）
5. 「**ウェブフック URL をコピー**」→ 控えておく

---

## STEP 2 — Apps Script にコードを貼り付ける

1. **返金ステータス管理シート**を開く
2. メニュー「**拡張機能**」→「**Apps Script**」
3. エディタが開いたら、既存のコードをすべて削除
4. `apps_script.js` の内容をまるごとコピー＆ペースト
5. 冒頭の `DISCORD_WEBHOOK_URL` に STEP 1 でコピーした URL を貼る

```javascript
var DISCORD_WEBHOOK_URL = "https://discord.com/api/webhooks/xxxxx/yyyyy";
```

6. 「**保存**」（Ctrl+S または 💾 アイコン）

---

## STEP 3 — 動作テスト

1. 関数の選択欄で **`sendRefundReport`** を選ぶ
2. 「**実行**」ボタンを押す
3. 初回は権限の確認ダイアログが出る → 「権限を確認」→ 自分の Google アカウントを選択 → 「許可」
4. Discord に通知が届けば成功！

---

## STEP 4 — 毎朝9時の自動実行を設定する

1. 関数の選択欄で **`setupTrigger`** を選ぶ
2. 「**実行**」ボタンを押す
3. ログに「✅ トリガー設定完了」と表示されれば完了

以降は毎朝9時に自動で Discord に送信されます。
（Mac がオフでも動きます — Google のサーバー上で実行されるため）

## トラブルシューティング

| 症状 | 対処法 |
|------|--------|
| 「シートが見つかりません」エラー | シート名が「ダッシュボード」と完全一致しているか確認 |
| Discord に届かない | Webhook URL が正しいか確認（`https://discord.com/api/webhooks/` で始まる） |
| 数値が 0 のまま | ダッシュボードの計算式が正しく動いているか確認 |
| 毎朝届かない | setupTrigger を再実行、または Apps Script の「トリガー」メニューで確認 |
