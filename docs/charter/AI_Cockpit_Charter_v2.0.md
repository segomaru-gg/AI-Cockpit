# AI-Cockpit Charter v2.0  
*(Machine-Readable Operational Spec)*

## 1. System Purpose

AI-Cockpit is a structured operating system for Sego’s multi-project management.

Primary objective:

Maintain structural coherence between philosophy, design, and execution.

Speed is secondary to coherence.

---

## 2. Structural Hierarchy

All projects MUST follow this hierarchy:

Project (no deadline)
 └─ QuarterGoal (has deadline, 3 months)
     └─ Phase (has deadline, sequential)
         └─ Task (no independent deadline)

---

## 3. Quarter Rules

### 3.1 Calendar Alignment

Quarter definitions are fixed:

Q1: Jan–Mar  
Q2: Apr–Jun  
Q3: Jul–Sep  
Q4: Oct–Dec  

No custom quarters allowed.

---

### 3.2 Quarter Theme

Each quarter has one global theme.

Theme properties:

- Has no completion criteria.
- Is not measurable.
- Influences decision direction only.
- Exists at AI-Cockpit root level.

Example (2026):

Q1: Transition  
Q2: Stabilization  
Q3: Execution  
Q4: Outlook  

---

## 4. Phase Framework (Global Standard)

All projects MUST use shared phase skeleton:

P0: Definition  
P1: Design  
P2: Implementation  
P3: Validation  
P4: Stabilization  

Phases are sequential by default.

Skipping phases requires explicit reasoning.

---

## 5. Goal Requirements

Each QuarterGoal MUST define:

- Explicit completion condition
- Deadline (within quarter)
- Measurable or verifiable outcome

Themes never contain completion conditions.

---

## 6. Task Rules

Tasks:

- Belong to one Phase only.
- Do not hold independent deadlines.
- Must produce a concrete artifact or state change.
- Should be 1–2 hour execution units.

---

## 7. AI Interaction Constraints

AI:

- May analyze
- May structure
- May propose
- May generate drafts

AI may NOT:

- Redefine structural hierarchy
- Modify directory architecture without approval
- Perform large-scale refactoring autonomously
- Create recursive automation loops

Sego retains final decision authority.

---

## 8. Change Control Protocol

Before any structural modification:

1. Explain intent.
2. Identify impact scope.
3. Limit change surface.
4. Confirm before execution.

---

## 9. System Philosophy Layer

AI-Cockpit enforces:

Philosophy → Structure → Execution order  
Small iterative stabilization  
Quarterly realignment with calendar time

---

## 10. Q1 2026 Special Context

Global Theme: Transition

Interpretation scope:

- Role transition
- Structural migration
- Institutionalization of informal systems

Theme influences evaluation but is not evaluated.
