## 🏗 何が変わったの？

これまでは、プロジェクト名がその場のノリやコードネームで付けられており、トップページ（ルートディレクトリ）にいろいろなファイルが散らばっている、「整理前の引越しダンボール」のような状態でした。

それを、**「整理整頓されたシステム手帳」**のような形に作り直しました。

### 1. 名前がわかりやすくなりました

暗号のようなプロジェクト名を、**「それが何をする場所なのか」**が一目でわかる名前に変更しました。

| 以前の名前 | 新しい名前 | 役割・意味 |
| :--- | :--- | :--- |
| **BluePrint** | 👉 **cockpit-core** | **コックピット（司令室）**。<br>全体のダッシュボードを表示する場所です。 |
| **Oduman** | 👉 **consulting** | **コンサルティング**。<br>ビジネス戦略やクライアントワークを管理する場所です。 |

### 2. 部屋（ディレクトリ）が整理されました

これまでは、玄関（トップフォルダ）を開けると、いきなり全ての部屋のドアが並んでいる状態でした。
新しい構造では、**`domains`（ドメイン＝領域）** という大きな「区分け」を作り、その中に各プロジェクトを収納しました。

**変更前（Before）:**
```text
AI-Cockpit/
├── BluePrint/   (ダッシュボード)
├── Oduman/      (ビジネス)
├── TOEIC/       (勉強)
├── BoaBASE/     (事務)
├── aeternum/    (ブランド)
├── .gitignore
└── README.md
```

**変更後（After）:**
```text
AI-Cockpit/
├── domains/               <-- 📂 全ての活動領域はここに収納！
│   ├── cockpit-core/      (司令室)
│   ├── consulting/        (ビジネス)
│   ├── operations/        (実務)
│   ├── learning/          (学習)
│   └── portfolio/         (ブランド)
├── docs/                  <-- 📝 説明書やマニュアル置き場
└── config/                <-- ⚙️ 将来のための設定ファイル置き場
```

## ✨ 何が良くなったの？

1.  **迷わなくなりました**
    *   「ビジネスの案件どこだっけ？ Odumanだっけ？」と悩む必要がありません。「consulting」を見ればそこにあります。
2.  **散らからなくなりました**
    *   今後新しいプロジェクト（例えば「趣味」や「健康管理」）が増えても、全て `domains` の中に入れれば良いので、トップページが散らかりません。
3.  **プロフェッショナルな見た目になりました**
    *   誰に見せても恥ずかしくない、論理的で美しい構造になりました。

## 🚀 新しい使い始め方

ダッシュボード（コックピット）を起動する場所が少しだけ変わりました。

1. **ターミナルを開く**
2. **以下のコマンドを入力**

```bash
# コックピットの部屋へ移動
cd domains/cockpit-core

# ダッシュボード起動
npm run dev
```

これだけで、今まで通り（いえ、今まで以上に快適に）システムを利用できます。
